# DressUpExample

Суть тестового задания:
Изучить игру-референс, чтобы изучить механику нанесения макияжа (крем, тени, помада)  
Реализовать игровую механику мейкапа с помощью подготовленных ресурсов, собрать билд и подготовить APK файл.

Подробное описание: 

Управление рукой осуществляется через Drag (onDrag) до зоны лица.
Переход контроля игрока и автоматических анимаций строго по зоне действия (зона лица).
Плавные переходы между состояниями без рывков и резких движений.
Взаимодействие с тремя предметами: крем, тени, помада, румяна. 

Механика нанесения крема (убираем прыщи) 
Пошаговая логика:
Игрок нажимает на объект «крем».
Запускается анимация «рука берёт крем».
После взятия:
Рука с кремом переходит в позиции, между лицом и местом, где ранее стоял крем
Управление переходит игроку: игрок самостоятельно перетаскивает руку с кремом к лицу.
Если игрок отпускает палец вне зоны лица, ничего не происходит 
Если игрок отпускает палец в зоне лица:
Запускается анимация «нанесение крема на лицо».
- Меняется спрайт с девочки с прыщами, на девочку без прыщей
После окончания анимации -  рука автоматически возвращает крем на место.
Рука в дефолтной позиции.

Механика нанесения теней (Наносим тени на куклу)
Игрок нажимает на объект «цвет теней».
Рука берёт кисточку и касается цвета в палетке, как бы нанося тени на кисточку (кончик кисточки красится в цвет, которую окунали кисточку) 
Рука фиксируется на уровне груди, кисточка смотрит вверх.
Управление передаётся игроку: ведёт к лицу руку с кисточкой.
Отпускает в зоне лица:
Запуск анимации «нанесение теней». Достаточно быстрая анимация (смотрим реф) 
- Наносим тени на персонажа
Рука возвращает кисточку и становится в дефолтное положение.

Механика нанесения помады (Красим губы) 
Игрок нажимает на помаду
Рука берёт помаду
Фиксируется на уровне груди.
Игрок ведёт к лицу, остальное как при тенях (меняем цвет губ) 

Дополнительные условия:
При нажатии на голубой спонжик макияж стирается автоматически без действия игрока и анимаций.
